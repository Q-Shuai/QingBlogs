Globax AI — 外贸 AI 工具站 项目详解

访问地址：https://globax-ai.vercel.app/login

预览效果：

一、项目定位

这是一个面向外贸（跨境电商/B2B）从业者的 AI 办公工具箱，帮助用户使用 AI（支持 OpenAI GPT-4o、Google Gemini、Anthropic Claude 及第三方中转 API）完成内容创作、关键词研究、广告文案生成等工作。

技术栈：React 19 + TypeScript + Vite 8 + React Router 7 + Supabase + CSS Modules

二、整体架构

目录结构职责

目录 职责
src/features/ 按功能模块组织的页面组件，每个模块 .tsx + .module.css
src/services/ 抽象服务层：AI 调用、搜索数据、Supabase 客户端、数据同步
src/context/ 全局状态管理（React Context），存储 API Keys、配置、用户状态
src/components/ 通用组件：侧边栏 Sidebar、认证守卫 AuthGuard
src/hooks/ 自定义 Hook：useLocalStorage（本地持久化）
src/utils/ 工具函数 + Prompt 模板

三、核心架构详解

3.1 全局状态管理 — AppContext.tsx

这是整个应用的”大脑”，管理以下状态：

  • API 密钥：OpenAI Key、Gemini Key、Claude Key、中转 API Key、Serper Key
  • 行业背景：多套”业务配置”（Business Profile），包含业务信息、典型客户
  • 首选模型：用户选择的默认 AI 模型
  • 自定义 Prompt：每个功能模块的提示词都可以在配置中心自定义
  • 主题/UI 偏好：亮色/暗色模式、侧边栏折叠状态
  • 认证状态：用户登录/注册/登出，用户画像信息
  • 云端同步：登录后将 LocalStorage 配置同步到 Supabase

关键设计：

  • 所有状态通过 useLocalStorage Hook 持久化到浏览器本地
  • 每个 setter 都包装了 pushToCloud()，修改配置时自动同步到 Supabase
  • industryBackground 是从活跃的业务配置中动态计算出来的字符串，注入到所有 AI Prompt 中

3.2 AI 调用层 — aiProvider.ts（最重要的服务）

这是项目的核心引擎，采用 策略模式 统一封装了 4 条 AI 调用通路：

关键函数：

  • callAI(prompt, systemPrompt, model) — 单轮对话（system + user）
  • callChat(messages, model) — 多轮对话（支持历史消息）
  • testConnection(provider, config) — 测试 API 连接
  • fetchModels(baseUrl, apiKey) — 拉取可用模型列表

特殊处理：

  • Gemini API 格式转换：将 OpenAI 的 messages 格式转为 Gemini 的 contents + system_instruction 格式
  • Claude 的 system prompt 被提到请求顶层而非放入 messages
  • Claude Key 未配置时，自动 fallback 到中转 API 调用 Claude 模型

3.3 数据持久化 — Supabase + dataSync

本地层：useLocalStorage Hook，自动将状态 JSON 序列化到 localStorage

云端层：Supabase 提供：

  • 认证：邮箱注册/登录，Session 管理
  • 配置同步：user_config 表（JSONB）存储完整的用户配置
  • 历史记录：6 张表存储各功能模块的历史数据

数据同步流程：

安全设计（RLS）：

  • 所有表启用 Row Level Security
  • 每个用户只能读写自己的数据（auth.uid() = user_id）
  • API Key 存储在客户端，Supabase 里存的也是用户自己的 Key

四、七大功能模块详解

4.1 关键词研究 (KeywordResearch.tsx)

流程：用户输入关键词（每行一个，最多100个） → 调用 Search API 批量查询 → 展示结果表格 + 趋势图 → 可导出 CSV

技术要点：

  • 调用第三方 API ke.octoo.top 获取搜索量、CPC、竞争度、12个月趋势
  • 并发请求（Promise.all），每个关键词独立请求
  • 趋势数据用 CSS 柱状图可视化（纯 CSS，无第三方图表库）
  • 竞争度用进度条展示
  • 登录用户自动保存查询记录到 Supabase

4.2 广告文案生成 (AdGenerator.tsx)

流程：用户填写行业关键词 + 目标市场 + 站内链接 → 选择模型 → 调用 AI → 解析 JSON → 展示 Headlines/Descriptions/Sitelinks/Callouts → 一键复制/导出 CSV

技术要点：

  • 使用结构化的 System Prompt（在 defaultPrompts.ts 中定义），要求 AI 严格按 JSON 格式返回
  • Prompt 中包含 Google Ads 的字符限制规则（标题30字符、描述90字符等）
  • 前端展示每个素材的字符数，方便验证
  • 支持从 JSON 中提取 {...} 块（容错解析）

4.3 博客文章生成 (BlogPostGenerator.tsx)

流程：用户输入标题 →（可选）副标题/见解/自定义要求 → AI 生成完整文章 → Markdown 渲染 → 可下载 .md

技术要点：

  • Prompt 设计遵循 EEAT（Experience, Expertise, Authoritativeness, Trustworthiness）标准
  • “反 AI 化”指令：禁止使用陈词滥调（如 “In today’s fast-paced world…”）
  • 自动从生成内容中提取 AI 生图提示词（支持 6 种 Markdown 格式）
  • 使用 markdown-to-jsx 渲染 Markdown 预览
  • 可将 AI 生图提示词单独提取并导出

4.4 Insight 共创 (InsightCoCreation.tsx)

这是最有特色的功能——AI 访谈员模式：

流程：

  1. 用户输入文章选题 → 选择 AI 模型 → 开启对话
  2. AI 以”访谈员”角色发起结构化提问（使用 /public/SKILL.md 中的方法论）
  3. 用户与 AI 多轮对话，逐步打磨思路
  4. 对话充分后，点击”生成最终 Insight” → AI 输出结构化的洞察文档

核心理念：AI 做 80% 的思考，用户做 20% 的选择。通过 Compress（压缩）、Negate（否定/反面）、Reconstruct（重构）三种方法将模糊想法变为精准洞察。

技术要点：

  • 多轮对话（callChat），保持完整上下文
  • 从 TopicManager 可跳转过来（携带话题标题）
  • 最终产物可生成到博客文章生成器

4.5 话题管理 (TopicManager.tsx)

流程：手动添加话题 / 从 Excel 批量导入 → AI 自动分类（B端/C端/BC端）→ 统计面板 → 跳转到 Insight 共创 / 博客生成

技术要点：

  • 使用 xlsx 库解析 Excel 文件
  • 导入时逐个调用 AI 进行 B/C 分类
  • 每个话题可关联 Insight，并支持跳转到其他功能模块（无缝衔接工作流）
  • 统计卡片：总话题数 / B端 / C端 / BC端 / 待分类

4.6 外链引用生成 (CitationGenerator.tsx)

流程：粘贴文章 → Step 1: AI 分析找出需要引用的观点 → Step 2: AI 搜索权威链接 → 自动插入引用标记 [^1] → 生成引用章节

技术要点：

  • 两步 AI 调用：先分析事实性声明（claim extraction），再匹配引用链接
  • 用户可以在配置中心设置”优先域名”（如 wikipedia.org）和”黑名单域名”（如 youtube.com）
  • 最终输出是增强后的 Markdown（原文 + 引用标记 + References 章节）

4.7 配置中心 (SettingsPage.tsx)

四个 Tab：

  • 行业背景：管理多套业务配置（Business Profile），选择启用哪一套 → 注入所有 AI Prompt
  • AI 配置：API Base URL + Key + 模型选择 + 连接测试 + 拉取模型列表
  • 外链引用设置：黑名单域名 + 优先域名
  • Prompt 模版：7 个功能模块的 System Prompt 全部可自定义编辑，支持变量占位符 {{variable}}

4.8 历史记录 (HistoryRecords.tsx)

统一的云端历史数据查看器，按功能模块分 Tab 展示，支持详情查看、单条/批量删除、分页。

五、数据流全景

六、关键设计亮点

  1. 统一 AI 接口 + 动态分发：4 条 AI 通路对上层透明，用户切换模型只需改变一个参数
  2. 可自定义 Prompt 系统：所有 System Prompt 都在配置中心可视化编辑，满足高级用户需求
  3. 行业背景注入：配置的行业信息自动注入所有 AI 调用，确保生成内容的行业相关性
  4. 本地优先 + 云端同步：未登录可用全部功能（数据存本地），登录后自动同步到云端
  5. 模块间联动：话题 → Insight 共创 → 博客生成，形成完整的内容生产流水线
  6. 纯前端架构：API Key 仅存客户端，零服务端存储敏感信息
  7. Vercel 部署就绪：配置了 SPA 路由重写，解决刷新 404 问题

七、部署方式

  • 开发：npm run dev（Vite 开发服务器）
  • 构建：npm run build（产出静态文件到 dist/）
  • 部署：dist/ 目录可部署到 Vercel、GitHub Pages 等任意静态托管平台
  • 当前已配置 vercel.json 的 SPA 路由重写规则

总结来说，这是一个设计精良的前端 AI 工具站，核心思路是用 AI 赋能外贸从业者的内容生产全流程——从关键词研究（数据驱动）→ 话题规划 → Insight 共创（深度思考）→ 博客/广告生成（内容产出）→ 引用增强（权威性），形成闭环。
