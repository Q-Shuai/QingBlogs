网页开发三件套(Html+CSS+Javascrit) + Tailwind 基础语法速览

如果你完全没碰过前端代码，看到 <div class=’…’> 这种符号就头大，那这篇就是为你写的。

我刚开始学建站时也一样，打开浏览器的开发者工具看着一堆代码发懵。后来发现，根本不需要把前端从头到尾学一遍——搞懂三件套各自管什么、基本语法长什么样，就已经能看懂 80% 的代码了。

这篇把 HTML、CSS、JavaScript、Tailwind 的基础语法拆开，用建站人的视角讲清楚。读完这篇，你再去读我写的实战篇（也就是上一篇文章），就不会觉得陌生了。

适用场景

这个教程解决什么问题：

  • 看到网页源代码一堆尖括号和花括号，完全不知道从哪看起
  • 想自己改点页面样式，但连 class 是什么意思都不清楚
  • 准备学 LiveCanvas，但担心自己不懂代码用不起来
  • 想知道 HTML、CSS、JS 到底谁管谁、怎么配合

谁适合看：零基础或只听过名词的建站新人。

建议先看这篇，再看《建站人员必修课：HTML+CSS+JS+Tailwind学习笔记（LiveCanvas 实战篇）》。两篇搭配着来，效果最好。

前置条件

  • 一个浏览器（Chrome 或 Edge 最好）
  • 打开任意一个网页，按下 F12，切换到 Elements 面板——你看到的那些代码就是 HTML
  • 愿意花 15 分钟看完这篇文章
  • 不需要任何编程基础

一、HTML：骨架的语言

标签长什么样

HTML 的核心就是标签。所有内容都用标签包起来。

<标签名>内容</标签名>

一个标签由三部分组成：

  • 开标签：<标签名>，告诉浏览器「这开始了」
  • 内容：标签里装的东西
  • 闭标签：</标签名>，告诉浏览器「这结束了」

拿现实例子来理解：

<p> 代表 paragraph（段落），<h1> 代表 heading（标题）。h 后面的数字越大，字号越小。

有些标签没有内容，叫自闭合标签：

<img src='图片地址.jpg' alt='描述文字' /> <br /> <!-- 换行 -->

属性的概念

标签还可以加属性，用来描述标签的额外信息：

<a href='https://google.com'>点我访问谷歌</a> <img src='product.jpg' alt='产品图片' />

这里的 href、src、alt 都是属性：

  • href：链接跳转到哪里
  • src：图片的来源路径
  • alt：图片的文字描述（加载不出来时显示）

格式固定：属性名=’属性值’（用空格隔开不同属性）

嵌套与层级

HTML 的标签可以套娃：

<div> 包着 <h2> 和 <p>，像俄罗斯套娃一样。这种关系叫父子关系。被包在里面的叫「子元素」，包在外面的叫「父元素」。

我刚开始总是搞混闭标签的顺序。记住一个口诀：先开的后闭。上面这个例子里，<div> 先开，那它就必须在最后才关闭。

盒模型——理解布局的基础

每个 HTML 元素在浏览器里都是一个盒子：

从内到外：

  • 内容：文字或图片
  • padding（内边距）：内容到边框的距离
  • border（边框）：边缘线
  • margin（外边距）：这个盒子到其他盒子的距离

这是我踩过的坑：一个 div 设了 width: 200px，再设 padding: 20px，实际占用的宽度是 200 + 20 + 20 = 240px！因为 padding 是在 width 外面扩张的。解决方法：加一句 box-sizing: border-box，告诉浏览器「width 给我包含 padding 和 border」。

常用 CSS 属性速查

属性 作用 示例
color 文字颜色 color: red;
font-size 字号 font-size: 16px;
background-color 背景色 background-color: #f0f0f0;
width / height 宽高 width: 100%;
margin 外边距 margin: 10px;
padding 内边距 padding: 20px;
border 边框 border: 1px solid black;
display 显示方式 display: flex;

这 8 个属性覆盖了日常 80% 的样式需求。

四、Tailwind：写样式的快捷方式

什么是 utility class

传统写 CSS 需要你做两步：先在 HTML 里给元素加 class 名，再到 CSS 文件里写样式。

Tailwind 换了个思路：把每个 CSS 属性组合都做成一个现成的 class，你直接在 HTML 上拼 class 就行。

p-4 就是 padding: 16px，bg-white 就是 background-color: white，rounded-lg 就是 border-radius: 8px。

Tailwind 的命名规则

Tailwind 的 class 名有规律可循，学会了就能猜：

前缀 含义 示例 解释
p- padding（内边距） p-4 四边内边距 16px
px- 左右 padding px-6 左右内边距 24px
py- 上下 padding py-3 上下内边距 12px
m- margin（外边距） m-2 四边外边距 8px
mt- / mb- / ml- / mr- 单个方向的 margin mt-4 上外边距 16px
text- 文字相关 text-lg 大号文字
bg- 背景相关 bg-blue-500 蓝色背景
flex / grid 布局模式 flex 弹性布局
rounded- 圆角 rounded-xl 大圆角
shadow- 阴影 shadow-md 中等阴影
hover: 鼠标悬停时 hover:shadow-lg 悬停时阴影变大
md: / lg: 屏幕尺寸断点 md:text-lg 平板以上文字变大

对照表：Tailwind 到原生 CSS

在 LiveCanvas 里直接用 Tailwind

我推荐在 LiveCanvas 里用 Tailwind，原因就一个——省事。

你用拖拽做出一个 section 后，如果想微调间距，不用打开 CSS 面板，直接在元素的 class 里加 p-6 或 mt-4 就行。LiveCanvas 搭配 WindPress插件，默认加载了 Tailwind 版本，所以你打开编辑器就可以直接用，什么额外配置都不用做。

三件套怎么配合（一张图看懂）

做个按钮来举例，你就明白 HTML + CSS + JS 怎么各司其职了：

  • HTML 负责造出这个按钮
  • Tailwind 负责让按钮变好看（颜色、间距、圆角）
  • JS 负责按钮被点击时做什么

总结

三句话总结这篇：

  1. HTML = 内容的结构和骨架，记住标签名 + 属性 + 嵌套就够用
  2. CSS / Tailwind = 外观和样式，Tailwind 就是「把 CSS 写成 class 名」，不用另写样式文件
  3. JavaScript = 交互和动作，新手记住 addEventListener + classList 就能做很多事情

这三个角色分工明确，各管各的。看代码的时候，先分清「哪个部分是 HTML？哪个是 CSS？哪个是 JS？」，就不会觉得乱。
